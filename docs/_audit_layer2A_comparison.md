## 1. Frequency-by-Frequency Assessment

Each of the 36 unique frequency strings found in Layer 1A is evaluated below against AIAG/IATF standards. Verdicts use:
- **VALID** -- Frequency is acceptable per AIAG/IATF with no changes needed
- **VALID-NORMALIZE** -- Frequency is acceptable per standards but should be normalized to a canonical string
- **CONDITIONAL** -- Frequency is acceptable only when used for the right classification and process type
- **FLAG** -- Frequency needs review; may be inadequate for the classification or process context it is applied to
- **INVALID** -- Frequency does not meet AIAG/IATF requirements for the context where it is used

### 1.1 EVENT-BASED Frequencies

#### #1: `Cada pieza` (100% inspection)
- **AIAG Assessment**: VALID
- **Standards Basis**: AIAG CP-1:2024 Section 4.1D lists "100% (every piece inspected)" as a fully valid frequency. IATF 16949 Section 5.2 recommends 100% or continuous for CC items.
- **Context Appropriateness**:
  - For visual inspection (OP 50, OP 80, OP 110): Fully appropriate. AIAG Section 7.4 confirms 100% visual is standard for final assembly and appearance items.
  - For CC items (Aspecto visual CC, Costura CC): Correct and required. IATF demands 100% or error-proofing for critical characteristics.
  - For standard items (rebabas, bordes): Acceptable but potentially over-inspecting standard characteristics. Not a non-conformity -- just a cost consideration.
- **Used By**: Telas Planas, Termoformadas, Armrest, Insert, Top Roll, Patagonia template, template library
- **Verdict**: VALID. No change needed.

#### #2: `Cada pieza (Pre-Lanzamiento)` (100% pre-launch)
- **AIAG Assessment**: VALID
- **Standards Basis**: AIAG CP-1:2024 Section 1.5 and IATF 16949 Section 2.2 state that pre-launch plans incorporate increased frequency. 100% during pre-launch is standard practice, especially for AP=M items that would normally sample.
- **Context**: Generated by default rules for AP=M during pre-launch phase.
- **Verdict**: VALID. Correct application of pre-launch enhanced frequency.

#### #3: `100%` (implied by sampleSize, not frequency)
- **AIAG Assessment**: VALID-NORMALIZE
- **Note**: This appears in sampleSize field rather than frequency field. Per AIAG CP-1:2024, sample size "100%" combined with a stated frequency is valid. However, if used in the frequency field itself, normalize to `Cada pieza` for consistency.
- **Verdict**: VALID-NORMALIZE. Use `Cada pieza` as the canonical frequency string.

#### #4: `Cada recepcion`
- **AIAG Assessment**: VALID
- **Standards Basis**: AIAG Section 4.1B lists "At lot change / At material batch change" as a valid event-based frequency. Incoming material inspection at each reception is standard practice per ISO 2859 / IATF 16949 Clause 8.4.2.
- **Context Appropriateness**:
  - For reception operations (OP 10): Correct. Checking every incoming lot/delivery is standard for raw material verification.
  - For SC items (Flamabilidad FMVSS 302): Acceptable if paired with certificate verification AND periodic lab testing. Certificate alone at reception is minimum; periodic lab re-test is recommended.
- **Used By**: Telas Planas, Termoformadas, Top Roll, Patagonia template
- **Verdict**: VALID.

#### #5: `Cada lote`

- **Context Appropriateness**:
  - For dimensional checks at lot level (OP 50, OP 70): Appropriate for standard characteristics.

## 2. Time-Based vs Event-Based Analysis

### 2.1 AIAG 2024 Position on Volume-Based Preference

The AIAG CP-1:2024 manual introduced a paradigm shift: frequency should ideally be **volume-based** (every N pieces) rather than purely time-based (every N hours). The core reasoning is that production rate varies, and time-based frequency provides inconsistent containment windows.

However, the manual does NOT prohibit time-based frequencies. It uses "every 2 hours" as an explicit example of valid frequency. The guidance is a preference, not a mandate.

### 2.2 Time-Based Frequencies That Should Be Reconsidered

| Current Frequency | Count in CPs | Process Context | Volume-Based Alternative | Recommendation |
|---|---|---|---|---|
| `Cada hora` | ~10 items | SPC weight, dimensional, dosificacion | `Cada 50 piezas` (assuming ~50 pcs/hr) | **OPTIONAL**. Add volume note in parentheses for auditor clarity. Changing is not required. |
| `Cada 2 horas` | ~3 items | Temperature (Patagonia), default rule | `Cada 100 piezas` (assuming ~50 pcs/hr) | **OPTIONAL** for temperature (process param). For product characteristics, consider volume-based. |
| `Cada turno` / `1x turno` / `Por turno` | ~15 items | Destructive tests, packaging, muestreo | `Cada lote` or `1 muestra cada turno` | **NO CHANGE** for destructive tests (frequency is limited by cost/feasibility, not containment logic). |
| `Inicio turno` / `Inicio de turno` | ~20 items | Setup verifications, machine params | N/A (event-based, not time-based) | **NO CHANGE**. These are actually event-based (triggered by shift changeover), not purely time-based. |

### 2.3 Assessment of "Cada 2 horas" for AMFE-Derived CP Items

Per the question: Is "Cada 2 horas" acceptable for AMFE-derived items in Control Plans per AIAG 2024?

**Answer: Yes, with conditions.**

1. **AIAG explicitly uses "every 2 hours" as an example** in the CP-1:2024 frequency column definition. It is a textbook-valid frequency.

2. **CQI-23 uses "once every 4 hours"** for part weight in injection molding. "Every 2 hours" is stricter than this industry benchmark.

3. **The conditions for acceptability are:**
   - The item must NOT be a Critical Characteristic (CC). For CC items, 100% or continuous is expected.
   - The risk (AP/severity from FMEA) must be justified. For AP=L items, 2-hour intervals are defensible.
   - The containment window must be documented. In 2 hours at ~50 parts/hour, ~100 parts are produced. The reaction plan must account for sorting/containing these 100 parts.
   - Evidence of actual compliance must exist (records showing checks performed every 2 hours).

4. **Where "Cada 2 horas" is currently used:**
   - Patagonia template M2: Temperatura vinilo/sustrato (SC) -- **Acceptable** if SPC is applied.
   - Default rule for AP=L, sev>=9 -- **Conditional**. If the resulting item is CC (which is unlikely for AP=L, but possible if customer-designated), then this is insufficient.

### 2.4 Summary: Time-Based Frequencies Are Acceptable

The Barack Mercosul CPs use a healthy mix of event-based (~55%), time-based (~25%), and hybrid (~20%) frequencies. This distribution is appropriate. The 2024 AIAG preference for volume-based does not require converting all time-based frequencies. The recommendation is to **add volume-based equivalents in parentheses** where practical (e.g., "Cada hora (~50 pzas)") for auditor transparency, but not to replace the time-based frequencies wholesale.

**CC Summary:**
- **COMPLIANT**: 8 items (all using `Cada pieza` / 100% visual or `Inicio + c/hora` with SPC)
- **FLAG**: 5 items
  - 3 items use `Inicio turno` only for CC dimensional characteristics
  - 2 items use `1x turno` for CC destructive tests

**CC Findings:**

**Finding CC-1 (HIGH):** Three CC dimensional characteristics (Armrest OP 50, Insert OP 60, Top Roll OP 10) use `Inicio turno` (shift start only) as their frequency. Per IATF 16949, a CC dimensional characteristic checked only at shift start has NO containment for the remaining 7+ hours of the shift. If the dimension drifts mid-shift, all parts produced until the next shift start are suspect.
- **Proposed correction**: Upgrade to `Inicio de turno + cada 2 horas` minimum, or `Inicio de turno + cada hora` with SPC (Xbar-R chart with Cpk >= 1.67 target).

**Finding CC-2 (MEDIUM):** Two CC items in Top Roll (adherencia film OP 30, resistencia soldadura OP 60) use `1x turno` for destructive tests. While destructive testing naturally limits frequency, IATF expects CC items to have the maximum feasible frequency. One test per shift for a CC item is borderline.
- **Proposed correction**: Increase to `Inicio de turno + cada 4 horas` (2 tests per shift minimum). If destructive testing cost is prohibitive, document a formal justification in the CP noting that (a) the process is stable with Cpk >= 1.67 historically, (b) non-destructive indicators (visual, force feedback) supplement the destructive test, and (c) the 1x/shift frequency is the maximum feasible given test cost.

## 6. Algorithmic Default Rules Assessment

### 6.1 Current Default Rules (from `controlPlanDefaults.ts`)

| Rule | AP | Severity | Phase | Generated Frequency | AIAG Assessment |
|---|---|---|---|---|---|
| R1 | H | any | any | `Cada pieza` | **CORRECT** |
| R2 | M | any | preLaunch | `Cada pieza (Pre-Lanzamiento)` | **CORRECT** |
| R3 | M | >= 9 | production | `Cada hora` | **ACCEPTABLE** |
| R4 | M | < 9 | production | `Cada turno` | **ACCEPTABLE** |
| R5 | L | >= 9 | any | `Cada 2 horas` | **CONDITIONAL** |
| R6 | L | >= 5 | any | `Cada turno` | **ACCEPTABLE** |

### 6.2 Rule-by-Rule Analysis

**R1: AP=H --> Cada pieza (100%)**
- **Assessment: CORRECT.**
- AP=H (High Action Priority) indicates the highest risk level. Per AIAG-VDA FMEA Handbook, AP=H items require immediate action. Per IATF 16949, high-risk items should have 100% or continuous monitoring.
- 100% inspection for all AP=H items regardless of severity is the right default. This aligns with the expectation that AP=H items are either CC-classified or require equivalent controls.

**R2: AP=M, any severity, preLaunch --> Cada pieza (Pre-Lanzamiento)**
- **Assessment: CORRECT.**
- AIAG CP-1:2024 Section 1.5 states that pre-launch plans incorporate increased frequency. IATF 16949 Section 2.2 Phase 2 (Pre-launch) explicitly requires "increased frequency and tighter controls."
- Upgrading AP=M items to 100% during pre-launch is standard practice and reflects the safe-launch philosophy.

**R3: AP=M, severity >= 9, production --> Cada hora**
- **Assessment: ACCEPTABLE.**
- AP=M with severity >= 9 means medium priority but high impact. Hourly checks represent a strong containment frequency (~50 parts/hour containment window).
- Per IATF CC/SC differentiation: severity >= 9 often corresponds to SC or CC classification. Hourly checks with SPC would satisfy SC requirements. For CC items, this may need to be upgraded to `Cada pieza` or `Inicio + c/hora` with SPC depending on the specific characteristic and customer requirements.
- **Potential issue**: The rule does not distinguish between CC and SC. A severity-9 item with CC classification should arguably default to `Cada pieza`, not `Cada hora`.

**R4: AP=M, severity < 9, production --> Cada turno**
- **Assessment: ACCEPTABLE.**
- AP=M with severity < 9 means medium priority, moderate impact. Once per shift is a reasonable baseline for these items.
- Per IATF 16949, this level of frequency is standard for non-special characteristics of medium risk.

**R5: AP=L, severity >= 9, production --> Cada 2 horas**
- **Assessment: CONDITIONAL.**
- AP=L with severity >= 9 is an unusual combination: low priority (low occurrence, good detection) but high severity. This can happen when a failure mode is rare but catastrophic if it occurs.
- "Every 2 hours" is valid per AIAG (used as an example in the manual) and represents a moderate containment window (~100 parts at 50/hr).
- **Concerns**:
  1. Severity >= 9 could indicate a safety-related characteristic. If the item is CC, 2-hour frequency is insufficient per IATF 16949.
  2. The AP is Low because occurrence and/or detection are favorable, but severity is inherent -- the rule should check whether the item has CC classification, not just AP.
  3. A severity-9 or severity-10 item should at minimum have documented justification for why 2-hour intervals are adequate.
- **Recommendation**: Add a CC/SC check. If the item is CC (regardless of AP), default to `Cada pieza`. If SC, `Cada 2 horas` with SPC is acceptable.

**R6: AP=L, severity >= 5, production --> Cada turno**
- **Assessment: ACCEPTABLE.**
- AP=L with severity 5-8 means low risk across all dimensions. Once per shift is a reasonable and common frequency for standard characteristics of low risk.
- No concerns for this rule.

### 6.3 Missing Default Rules

The current rule set has gaps:

| Condition | Current Behavior | Recommended Default |
|---|---|---|
| AP=L, severity < 5 | No rule (fallback?) | `Cada turno` or `Cada lote` (low risk, low impact) |
| AP=H with CC classification | `Cada pieza` | `Cada pieza` + ensure CC is flagged for Poka-Yoke consideration |
| AP=M, severity >= 9, CC | `Cada hora` | Should be `Cada pieza` for CC, `Cada hora` only for SC |

### 6.4 Proposed Corrected Default Rules

```
R1:  AP=H, any severity, any phase        --> Cada pieza
R2:  AP=M, any severity, preLaunch         --> Cada pieza (Pre-Lanzamiento)
R3a: AP=M, severity >= 9, CC, production   --> Cada pieza              [NEW]
R3b: AP=M, severity >= 9, SC/Std, production --> Cada hora
R4:  AP=M, severity < 9, production        --> Cada turno
R5a: AP=L, severity >= 9, CC, production   --> Cada pieza              [NEW]
R5b: AP=L, severity >= 9, SC/Std, production --> Cada 2 horas
R6:  AP=L, severity >= 5, production       --> Cada turno
R7:  AP=L, severity < 5, production        --> Cada lote               [NEW]
```

The key change is introducing CC/SC awareness: **any CC item, regardless of AP level, should default to `Cada pieza` (100%)**. This aligns with the IATF 16949 requirement that CC monitoring be "relevant to the seriousness of the characteristic" and that CC items typically require 100% or continuous monitoring.
